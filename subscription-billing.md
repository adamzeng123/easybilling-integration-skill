# EasyBilling Subscription Billing

Flat-fee and tiered subscription workflows. Prerequisite: auth configured (see auth-setup.md).

## Step 1: Create an Account

```http
POST /api/accounts
Content-Type: application/json
Authorization: Bearer <token>

{
  "name": "Acme Corp",
  "accountNumber": "ACC-ACME-001",
  "email": "billing@acme.com",
  "addressLine1": "123 Main St",
  "city": "San Francisco",
  "state": "CA",
  "country": "US",
  "postalCode": "94105",
  "source": "website"
}
```

`accountNumber` is optional — omit it and EasyBilling auto-generates one. Save the returned `number` for all subsequent calls.

## Step 2: Create a Contract

All contract mutations go through a single endpoint:

```http
POST /api/contract-actions
Content-Type: application/json
Authorization: Bearer <token>

{
  "accountNumber": "ACC-ACME-001",
  "actions": [
    {
      "type": "create-contract-with-plan",
      "createContractWithPlan": [
        {
          "planId": "<plan-uuid>",
          "effectiveDate": "2026-05-01",
          "expirationDate": "2027-05-01",
          "currency": ["USD"]
        }
      ]
    }
  ]
}
```

The response includes `contractInfo` with `contractNumber` and `contractSegments`.

## Pricing Models

Override default plan pricing via `contractPrice` in the create action:

| Format | Use case |
|--------|----------|
| `per-unit` | Price per unit consumed (e.g., $0.01/API call) |
| `flat-fee` | Fixed price regardless of usage |
| `per-block` | Price per block of units (e.g., $10 per 1000 calls) |

Each pricing table has tiers with `start`, `end`, `price`, and optional `floorPrice`/`ceilingPrice`.

## Discounts

Add to `contractPrice.discounts`:
```json
{ "code": "LAUNCH20", "model": "percentage", "value": 20 }
```

## Contract Lifecycle Actions

All use `POST /api/contract-actions` with different `type` values:

| Action | Type | Required Fields |
|--------|------|-----------------|
| Switch plan | `switch-plan` | contractId, planId, effectiveDate, currency |
| Renew | `renew` | contractId, effectiveDate, expirationDate |
| Early renew | `early-renew` | contractId, planId, effectiveDate, expirationDate, currency |
| Update price | `update-price` | contractId, contractSegmentId, planId, contractPrice, effectiveDate |
| Cancel | `cancel` | contractId, effectiveDate |

## Reading Invoices

Invoices are generated by the billing engine — you read them, never create them.

```http
GET /api/invoices?accountNumber=ACC-ACME-001
Authorization: Bearer <token>
```

For line-item details:
```http
GET /api/invoices/<invoiceId>
```

Invoice statuses: `draft` → `posted` → `partial-paid` / `paid` / `cancelled`

## Pagination

List endpoints use cursor-based pagination:
```
GET /api/accounts?cursor=<cursor>&pageSize=100
```
Response includes `hasMore` (boolean) and `cursor` (string for next page). `pageSize` is effective when > 50.
