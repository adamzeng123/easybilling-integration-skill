## Key Rules

1. **Account first** вЂ” create before any other operation. Everything references `accountNumber`.
2. **Contracts are created via ContractActions** вЂ” `POST /api/contract-actions`, not a direct contract endpoint.
3. **ContractSegments are immutable** вЂ” plan switches, renewals, and price updates create new segments.
4. **Invoices are read-only** вЂ” the billing engine generates them. You query, never create.
5. **PaymentHub is separate** вЂ” payments link to billing via `transactionId`, not directly to invoices.

## Entity Lifecycle States

| Entity | States |
|--------|--------|
| Account | `active`, `in-active`, `draft` |
| ContractAction | `active`, `expired`, `canceled` |
| Invoice | `draft`, `posted`, `partial-paid`, `paid`, `cancelled` |
| Bucket | active/expired (by `validFrom`/`validTo`) |
| Transaction | `pending`, `processing`, `succeeded`, `failed`, `expired`, `closed` |
| Refund | `created`, `refunding`, `succeeded`, `failed`, `canceled`, `closed` |

## ID Formats

- **Internal IDs:** UUIDs (e.g., `e3ef9de3-bbdc-495c-b608-a6fa1f62ada0`)
- **Account numbers:** `ACC-XXXXXXXXXX` (auto-generated or custom)
- **Contract numbers:** `CT-XXXXXXXXXX`
- **Invoice numbers:** `INV-XXXXXXXXX`
- **Contract action numbers:** `CA-XXXXXXXXXX`
